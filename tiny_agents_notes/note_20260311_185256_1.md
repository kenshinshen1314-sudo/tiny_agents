## 代码质量问题

### 1. 异常处理不规范 (高优先级)
**位置**: 
- tools/builtin/file_tools.py (多处)
- tools/builtin/note_tool.py
- memory/types/semantic.py
- memory/storage/qdrant_store.py
- memory/storage/neo4j_store.py

**问题**: 使用裸 `except:` 语句，未指定异常类型
```python
except:
    pass
```

**风险**: 
- 隐藏所有异常，难以调试
- 可能掩盖严重错误
- 违反 PEP 8 规范

**建议**: 捕获具体异常类型，如 `except (KeyError, ValueError) as e:`

---

### 2. 调试日志未清理 (中优先级)
**位置**: memory/types/semantic.py
- 13 处 `logger.info(f"🐛 [DEBUG] ...")`

**问题**: 
- 调试日志使用 info 级别
- 包含 emoji 标记 🐛
- 生产环境会产生大量日志

**建议**: 
- 使用 `logger.debug()` 替代
- 移除 emoji 标记
- 或添加配置开关控制
