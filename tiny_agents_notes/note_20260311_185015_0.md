# tiny_agents 项目架构概览

## 项目基本信息
- 项目名称: tiny_agents (轻量级 AI Agent 框架)
- 版本: v0.1.0
- Python 要求: >=3.8
- 核心依赖: openai>=1.0.0, python-dotenv>=1.0.0

## 项目结构

### 核心模块 (tiny_agents/)
1. **core/** - 核心框架
   - agent.py (43KB): Agent 基类，集成历史管理、Token计数、可观测性
   - llm.py (14KB): LLM 封装
   - config.py: 配置管理
   - database_config.py: 数据库配置
   - lifecycle.py: 生命周期管理
   - session_store.py: 会话存储
   - streaming.py: 流式输出
   - message.py: 消息定义
   - exceptions.py: 异常定义

2. **agents/** - Agent 实现
   - function_call_agent.py (14KB): 函数调用 Agent
   - react_agent.py (6.6KB): ReAct Agent
   - plan_solve_agent.py (6.7KB): 规划求解 Agent
   - reflection_agent.py (5.8KB): 反思 Agent
   - simple_agent.py (2.5KB): 简单 Agent

3. **tools/** - 工具系统
   - base.py (14KB): 工具基类
   - registry.py (10KB): 工具注册表
   - chain.py (5KB): 工具链
   - circuit_breaker.py (4.9KB): 熔断器
   - tool_filter.py (4.3KB): 工具过滤
   - response.py (4.7KB): 工具响应
   - errors.py (2KB): 错误定义
   - async_tool_executor.py (2.3KB): 异步执行器
   - builtin/**: 内置工具集
     - search.py, search_tool.py: 搜索工具
     - file_tools.py (26KB): 文件操作工具
     - memory_tool.py (19KB): 记忆工具
     - note_tool.py (17KB): 笔记工具
     - rag_tool.py (34KB): RAG 工具
     - terminal_tool.py (7KB): 终端工具
     - devlog_tool.py (13KB): 开发日志工具
     - calculator.py, task_tool.py, skill_tool.py, todowrite_tool.py

4. **memory/** - 记忆系统
   - manager.py (11KB): 记忆管理器
   - base.py (4KB): 记忆基类
   - embedding.py (11KB): 向量嵌入
   - storage/**: 存储后端
   - rag/**: RAG 实现
   - sessions/**: 会话存储
   - traces/**: 追踪数据
   - types/**: 记忆类型定义

5. **context/** - 上下文管理
   - builder.py (14KB): 上下文构建器
   - history.py (4.8KB): 历史管理
   - token_counter.py (4.3KB): Token 计数
   - truncator.py (5.4KB): 输出截断

6. **skills/** - 技能系统
   - loader.py (6KB): 技能加载器

7. **observability/** - 可观测性
   - trace_logger.py (16KB): 追踪日志 (JSONL + HTML)

8. **assistant/** - 助手实现
   - CodebaseMaintainerAssistant/
   - LearningAssistant/

### 其他目录
- tests/: 测试代码
- demo/: 示例代码
- docs/: 文档 (包含测试调用链文档)
- tiny_agents_notes/: 笔记
- knowledge_base.json: 知识库
- memory_data/: 记忆数据

## 核心特性
1. 多种 Agent 模式 (ReAct, Plan-Solve, Reflection, Function Call)
2. 完整的工具系统 (注册、过滤、链式调用、熔断)
3. 记忆系统 (向量检索、RAG、多类型记忆)
4. 可观测性 (追踪日志、HTML 可视化)
5. 上下文工程 (历史压缩、Token 管理、输出截断)
6. 技能系统 (知识外化)
7. 流式输出支持
8. 异步执行支持

## 设计亮点
- 模块化设计，各组件职责清晰
- 支持工具注册和自动发现
- 完善的配置系统
- 良好的错误处理和日志
- 支持生命周期钩子
- 向后兼容性设计
